Задание: Написать и обосновать в комментарии минимальное количество тест кейсов для тестирования примера из первого урока данного модуля (слайд Кейс применения классов эквивалентности и техники граничных значений).


Тесты класса эквивалентности

1. Проверка значения до 1000$
2. Проверка значения в 1000$
3. Проверка значения от 1000$ до 2000$
4. Проверка значения в 2000$
5. Проверка значения выше 2000$

Минимальное количество тестов которое можно создать по Классам эквивалентности 5, в данном примере получаем проверки: любое значение до 1000, проверка значения 1000, любое значени от 1000 до 2000, проверка значения 2000 и любое значение выше 2000.

Тесты граничного значения

1. Проверка значения 999
2. Проверка значения 1000
3. Проверка значения 1001
4. Проверка значения 1999
5. Проверка значения 2000
6. Проверка значения 2001

Для проверка граничных значений минимальное количество тестов получилось 6, где мы проверяем граничные значения числа 1000 (получаем значения 999 и 1001) и проверяем граничные значения числа 2000 (получаем значения 1999 и 2001)
